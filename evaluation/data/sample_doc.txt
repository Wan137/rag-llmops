Introduction to RAG Systems

Retrieval-Augmented Generation (RAG) is an AI architecture that combines
information retrieval with large language model generation. Unlike standard
LLMs that rely solely on parametric knowledge, RAG systems answer questions
grounded in a specific document corpus provided at runtime.

How Chunking Works

Text chunking is the process of splitting long documents into smaller,
overlapping segments. The overlap ensures that context near chunk boundaries
is preserved. A typical configuration is 512 characters per chunk with a
64-character overlap.

RecursiveCharacterTextSplitter first tries to split on paragraph breaks,
then sentence endings, then individual words, and only falls back to
character-level splits as a last resort. This preserves semantic coherence
far better than fixed-size splitting.

Embedding and Retrieval

Each chunk is encoded into a high-dimensional vector by an embedding model.
At query time the user's question is embedded with the same model, and the
nearest chunks are retrieved via cosine similarity. These top-k chunks form
the context window passed to the LLM, which generates a grounded answer.

Evaluation with RAGAS

RAGAS provides automated, reference-free metrics for RAG pipelines.
Faithfulness measures whether every claim in the answer is supported by
the retrieved context - this is the primary guard against hallucinations.
Answer relevancy measures whether the answer addresses the actual question.
Context precision and context recall evaluate retrieval quality independently
of the generation step.
